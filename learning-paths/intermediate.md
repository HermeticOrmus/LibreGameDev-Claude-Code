# Intermediate Learning Path - Game Development

## Overview

This intermediate path introduces Game Development concepts and practices, building from fundamentals to practical application. Each module combines theory with hands-on exercises using Claude Code plugins.

## Prerequisites

- Completed the beginner path\n- Basic game project completed\n- Familiarity with GDScript or C#

## Modules

### Module 1: State Machines & Game AI

#### Concepts
- Finite state machines for characters\n- Behavior trees for NPC AI\n- Navigation and pathfinding\n- Enemy behavior patterns\n- AI decision making

#### Hands-On Exercise
Using the relevant plugin, practice the core concepts from this module. Start with a simple project and gradually increase complexity.

```
# Example workflow
1. Set up your environment
2. Create a minimal working example
3. Iterate and improve
4. Document what you learned
```

#### Key Takeaways
- Understand the fundamental concepts of State Machines & Game AI
- Know when and why to apply these concepts
- Be able to create basic implementations

---

### Module 2: Physics & Particle Systems

#### Concepts
- RigidBody physics in depth\n- Custom collision layers\n- Raycasting and area detection\n- Particle emitters and effects\n- Visual polish techniques

#### Hands-On Exercise
Build on Module 1 by integrating more advanced concepts. Focus on quality and best practices rather than speed.

```
# Example workflow
1. Review your Module 1 work
2. Identify improvement opportunities
3. Apply new patterns and techniques
4. Validate results
```

#### Key Takeaways
- Integrate multiple concepts effectively
- Apply best practices consistently
- Evaluate and improve existing work

---

### Module 3: Audio & UI Systems

#### Concepts
- Audio bus architecture\n- Dynamic music systems\n- Game UI/HUD patterns\n- Inventory and menu systems\n- Dialogue and cutscene systems

#### Hands-On Exercise
Combine all concepts from this path into a comprehensive project that demonstrates your understanding.

```
# Example workflow
1. Plan your comprehensive project
2. Apply all learned patterns
3. Test and validate thoroughly
4. Present and document results
```

#### Key Takeaways
- Synthesize concepts across modules
- Make informed decisions about approaches
- Create production-quality implementations

## Assessment

To verify mastery of this intermediate path:
- [ ] Complete all three module exercises
- [ ] Build a project combining concepts from all modules
- [ ] Review your work against the patterns in the skill files
- [ ] Identify areas for further learning

Continue to the **advanced** path for ECS architecture, multiplayer networking, procedural generation, and shader programming.
